## Необходимые шаги для полной работы системы

### Шаг 1: Создать таблицы в Supabase

1. Откройте **Supabase Dashboard** вашего проекта
2. Перейдите в **SQL Editor**
3. Скопируйте SQL из файла `DATABASE_SETUP.md`
4. Выполните queries в следующем порядке:
   - CREATE TABLE clubs
   - CREATE TABLE directions  
   - CREATE TABLE sections
   - CREATE TABLE users

**Пример SQL:**
```sql
CREATE TABLE clubs (
  id BIGSERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL UNIQUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO clubs (name) VALUES 
  ('Клуб А'),
  ('Клуб Б'),
  ('Клуб В');
```

## Безопасность и улучшения для продакшена

### ⚠️ Важные замечания:

1. **Хеширование паролей**: Текущая реализация использует простой SHA256. Для продакшена используйте:
   ```bash
   pnpm add bcryptjs
   ```
   И обновите `app/api/auth/register/route.ts` и `app/api/auth/login/route.ts`

2. **Секретные переменные**: Убедитесь, что все sensitive данные находятся в `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

3. **HTTPS**: На продакшене используйте HTTPS для защиты сессий

4. **CORS**: Если фронтенд и бэкенд на разных доменах, настройте CORS

## API Endpoints

### Аутентификация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `POST /api/auth/logout` - Выход

### Отчеты
- `GET /api/report?period=2024-01` - Загрузить отчеты
- `POST /api/submit` - Сохранить отчет
- `GET /api/export?period=2024-01` - Экспорт (серверная сторона)

### Опции комбо-боксов
- `GET /api/combo-options?type=clubs` - Загрузить клубы
- `GET /api/combo-options?type=directions` - Загрузить направления
- `GET /api/combo-options?type=sections&direction=КДН` - Загрузить секции
- `POST /api/combo-options` - Добавить новое значение
